benchzoo / locust sample target

This page is a static HTTP target served by a minimal nginx instance on port 8080. It exists solely to give Locust a stable endpoint to benchmark while benchzoo captures Locust's CSV output for parser development.

Why this page exists

Unlike synthetic benchmarks that time a sleep or a CPU loop, Locust fundamentally measures real HTTP request/response latency under a simulated user workload. It needs something to talk to. Rather than pointing Locust at an external service (unstable, rate-limited, noisy), we stand up a minimal nginx instance serving this page, hit it locally over the loopback interface, and record Locust's CSV reports.

What Locust actually measures here

Per-endpoint request counts, failure counts, response-time percentiles (p50 through p100), average content size, and aggregate throughput — the numbers Locust writes into its _stats.csv, _failures.csv, and _stats_history.csv files. The measurements are a property of nginx + Locust + the runner's kernel, not of this page's content. The page exists to be served, not to be scrutinized.

Why the page is not empty

Locust's default HTTP request is a simple GET and the response needs a non-trivial body for the average-content-size number to be interesting. A few paragraphs of prose is enough to make that metric non-zero without inflating page size to the point where the transfer rate dominates the latency measurement. The goal is a realistic-ish static page, not a stress test of nginx's sendfile path.

What this page is not

This is not a meaningful web application. There are no scripts, no external assets, no cookies, no forms. A real web-performance benchmark would look completely different (and Lighthouse, not Locust, is the right tool for that). This is an HTTP load-test target, which is a different question entirely.
