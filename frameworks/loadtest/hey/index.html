benchzoo / hey sample target

This page is a static HTTP target served by a minimal nginx instance on port 8080. It exists solely to give hey a stable endpoint to benchmark while benchzoo captures hey's text output for parser development.

Why this page exists

Unlike synthetic benchmarks that time a sleep or a CPU loop, hey fundamentally measures real HTTP request/response latency. It needs something to talk to. Rather than pointing hey at an external service (unstable, rate-limited, noisy), we stand up a minimal nginx container serving this page, hit it locally over the loopback interface, and record hey's report.

What hey actually measures here

Summary statistics (total, slowest, fastest, average, requests/sec), a response-time histogram bucketed across the observed range, a latency distribution at 10%/25%/50%/75%/90%/95%/99%, and a status-code distribution — the numbers hey prints at the end of every run. The measurements are a property of nginx + hey + the runner's kernel, not of this page's content. The page exists to be served, not to be scrutinized.

Why the page is not empty

hey's default HTTP request is a simple GET and the response needs a non-trivial body for the size/request and transfer numbers to be interesting. A few paragraphs of prose is enough to make those metrics non-zero without inflating page size to the point where transfer dominates the latency measurement. The goal is a realistic-ish static page, not a stress test of nginx's sendfile path.

What this page is not

This is not a meaningful web application. There are no scripts, no external assets, no cookies, no forms. A real web-performance benchmark would look completely different (and Lighthouse, not hey, is the right tool for that). This is an HTTP load-test target, which is a different question entirely.
