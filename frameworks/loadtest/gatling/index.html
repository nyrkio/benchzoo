benchzoo / gatling sample target

This page is a static HTTP target served by a minimal nginx instance on port 8080. It exists solely to give Gatling a stable endpoint to benchmark while benchzoo captures Gatling's simulation.log for parser development.

Why this page exists

Unlike synthetic benchmarks that time a sleep or a CPU loop, wrk fundamentally measures real HTTP request/response latency. It needs something to talk to. Rather than pointing wrk at an external service (unstable, rate-limited, noisy), we stand up a minimal nginx container serving this page, hit it locally over the loopback interface, and record wrk's report.

What wrk actually measures here

Latency percentiles (p50, p75, p90, p99), requests per second, and transfer rate — the numbers wrk prints at the end of every run. The measurements are a property of nginx + wrk + the runner's kernel, not of this page's content. The page exists to be served, not to be scrutinized.

Why the page is not empty

wrk's default HTTP request is a simple GET and the response needs a non-trivial body for the transfer/sec number to be interesting. A few paragraphs of prose is enough to make that metric non-zero without inflating page size to the point where the transfer rate dominates the latency measurement. The goal is a realistic-ish static page, not a stress test of nginx's sendfile path.

What this page is not

This is not a meaningful web application. There are no scripts, no external assets, no cookies, no forms. A real web-performance benchmark would look completely different (and Lighthouse, not wrk, is the right tool for that). This is an HTTP load-test target, which is a different question entirely.
