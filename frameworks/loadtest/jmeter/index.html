benchzoo / JMeter sample target

This page is a static HTTP target served by a minimal nginx instance on port 8080. It exists solely to give Apache JMeter a stable endpoint to benchmark while benchzoo captures JMeter's CSV output for parser development.

Why this page exists

Unlike synthetic benchmarks that time a sleep or a CPU loop, JMeter fundamentally measures real HTTP request/response latency. It needs something to talk to. Rather than pointing JMeter at an external service (unstable, rate-limited, noisy), we stand up a minimal nginx container serving this page, hit it locally over the loopback interface, and record JMeter's CSV results file.

What JMeter actually measures here

Per-sample timings (one row per HTTP request) including elapsed time, latency, connect time, response code, and bytes transferred. Unlike wrk, JMeter does not pre-aggregate into percentiles — the parser does that, grouping by the sample label ("homepage" in our test plan).

What this page is not

This is not a meaningful web application. There are no scripts, no external assets, no cookies, no forms. A real web-performance benchmark would look completely different (and Lighthouse, not JMeter, is the right tool for that). This is an HTTP load-test target.
